API新续租完整说明

👍 跟租号玩官网新续租保持一致,续租订单进行拆单展示。

需要说明订单展示、下单、结算、投诉各个环节新续租拆单的兼容点。

业务流程

1. API侧新增新续租接口(租号玩增加)

a. 用戶点击续租按钮,通过续租接口提交请求后,生成新的续租订单。

b. 老续租暂时保留,等到全部渠道切换到新续租后,完成接口下线。

## Page 2

c. 新续租接口需要区分端手游接口,接口说明需要包含请求参数、响应结果、请求示例等。

d. 新续租接口需要传递套餐字段,新续租支持套餐续租

2. 渠道用戶选择续租,渠道客戶端请求API续租接口发起续租请求(渠道侧交互限制)

a. 租客只能有一个未开始的续租单

b. 投诉中状态不允许下续租单

3. API接口接收续租请求,区分新旧续租接口(API新增新续租接口)

a. 新续租接口转发到新续租服务进行续租,旧续租接口转发到旧续租服务进行续租。

b. 租客被号主拉黑后,不允许续租

c. 续租单时间与货架的不可租时间发生冲突时,续租失败

4. 续租成功后,返回续租单信息

续租成功后的信息,需要渠道侧客戶端进行展示处理。

a. 续租单为新订单,有独立的订单号和解锁码

b. 续租单之间解锁码通用。实现的方式为:租客用系列订单中的任一解锁码进行上号,当前进行
的是系列续租单中的哪个订单,就展示哪个订单的订单信息,并标示“续租”信息供租客查看

c. 续租单状态分类:

▪ 续租单未开始时的状态为“已预约”

▪ 未开始即取消的状态为“取消预约”

▪ 开始后的状态为“进行中”

▪ 撤单投诉完成的状态为“已撤单”

▪ 正常结算的状态为“已完成”

5. 进入到续租订单时间时,需要做续租订单兼容处理

◦ 续租成功,第一个订单时间自动过渡至第二个订单,无需重新登录。。

6. 续租订单撤单

租客端撤单逻辑:

a. 整体撤单逻辑:撤单或取消续租单时,从要撤单或取消续租的订单开始,该系列订单往后的所
有订单都取消续租

b. 正在进行的订单撤单,按照投诉类型正常走撤单核实,按照扣款规则计算,最多扣至此笔订单
全部扣完,后续续租单不会因撤单订单扣款造成撤单的扣款

c. 免费撤单次数逻辑:API侧接口需要上报的userid和payid综合判断唯一用戶,根据用戶取消次
数来判断是否能够继续免费取消。(租号玩主站后台配置兼容免费取消次数和违约金比例)
