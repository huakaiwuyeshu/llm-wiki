API对接流程

对接流程

1. 商务确认对方合作意向,

2. 确认完成后,产品介入进行前期沟通过

a. 确认合作的方式

i. 关注合作的平台(对方的接入平台)

ii. 接入类型-端游/手游,是否具有客戶端(PC/APP)

iii. 确认对方是否可以完成全流程的租号上号业务

iv. 告知对方需要准备的产品/技术资源

v. 沟通支付问题和客服问题

3. 以上没有问题后,确认合作的具体内容

a. 确认对方开通的游戏、上号方式

b. 货架数据对接方案

4. 提供API文档开始学习

5. 同时对方提供前期准备资料

a. 提供未在租号玩注册的手机号

i. 首先去网吧后台进行注册,获取对应的网吧id(加价使用)(商务操作)

• 在API那个账戶下创建一个一级代理即可实现

ii. 之后将该账号升级为商戶身份(韩朋操作)

b. 移动端对接时,需要对方先提供包名和签名(SHA-1形式),同时产品根据对方公司名称,生成一
个业务名

6. 联系移动端表哥根据包名和签名,生成对应的
appid
和
appsecret

7. 并行进行任务:

a. 移动端生成SDK包(aar格式)完成上号能力支撑

b. PHP后台进行合作方配置上线,配置鉴权让合作方后续可以完成接口调用

8. 根据对方对接进度,进行技术支持对接
