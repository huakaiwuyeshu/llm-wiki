## 核心内容

### 导语:先破误解
**一个固定流程,不会因为中间接了大模型,就自动变成智能体;会调用工具,也不等于就是 Agent。**Agent 不是固定形态的产品,而是从早期工具调用一步步长出记忆、检索、动手、编排、协作、护甲等能力,最终成为可以沉淀经验的业务系统。读法:看每一步是被上一代什么短板逼出来的,走完自然看懂为什么落到「混合架构」。

### 01 主线:九步演进(一行式时间轴,右列 = 手册模块映射)
1. **LLM**(万能百科,地基):能理解和生成,但无记性、知识静态、不会动手
2. **记忆 Memory**(知识层,→手册 #10):补"每次调用都失忆"——短期上下文 + 长期持久化
3. **RAG**(知识层,→手册 #7):补"不懂私有业务、还会编"——先检索再回答,没依据明说未找到
4. ★ **Function Call & MCP**(能力层,→手册 #8):补"只会说不会做"——LLM 输出调用指令、代码执行;**决策与执行分离是安全可控的根**
5. ★ **Agent Loop / ReAct**(质量层,→主链路):补"多步任务谁来串"——想→做→看→再决定;金句:**Agent 不是一种产品形态,而是一种控制策略——就是这个循环本身**
6. **Skill**(能力层,→手册 #6):补"工具一多就选不准"——渐进式加载 SOP + 工具,按手册操作而非临场猜
7. **Multi-Agent**(质量层,→手册 #6):补"单个脑子带不动"——专家协作;成本翻倍,不到必要不拆
8. ★ **Harness**(质量层,→#8/#11/#12):补"上线就崩"——容错、限流、工具治理、人工审核、可观测;**从「能跑」到「能用」的分水岭**
9. **Claw**(形态):部署形态演进——Agent 跑到你电脑上,直接动手并验证,闭合决策-执行-验证

### 02 归纳:三层框架
- 第一层·知道什么 = 知识(②记忆 ③RAG)
- 第二层·能做什么 = 能力(④Function Call & MCP ⑥Skill)
- 第三层·怎么做得好 = 质量(⑤Agent Loop ⑦Multi-Agent ⑧Harness)

### 03 换个粗看法:按「自主度」四档选型(= 手册原图 2)
| 档 | 名称 | 能力 | 适用 |
|---|---|---|---|
| 第 1 档 | 早期 Agent | 聊天 + 工具调用 | 短链路任务:问一句、调个工具、答一句 |
| 第 2 档 | 工作流 Agent | 流程编排 | 企业稳定流程:步骤已知,按图执行,可控 |
| 第 3 档 | 自主 Agent | 规划 + 执行 | 开放复杂任务:自己拆解规划,灵活但要兜底 |
| 第 4 档 | 自进化 Agent | Skill + Memory | 沉淀经验成资产,越用越懂业务 |

桥接话术(化解"两张图对不上"):同样是「记忆」「Skill」,九步里靠前(按**能力补齐先后**排),四档里靠后(按**自主度高低**排)——轴不一样位置自然变,排法不同不是矛盾。但两种看法都只回答"能力越来越多",没回答最关键一问:**这些能力的控制权该归谁?**

### 04 真正决定架构的暗线:控制权钟摆(★ 本版关键新增)
早期·全用规则(死板,稍变就改代码)→ 自主 Agent·全交 LLM(灵活但失控跑偏、上线就崩;金句:**业务最怕的不是系统不够聪明,而是系统自作聪明**)→ 工程实践·收回控制权(确定部分代码/流程固化,只在需要判断处放 LLM)。
收敛:**「Workflow 负责边界,Agent 负责弹性」**——Workflow 规定哪些步骤必须走、哪些权限不能越、哪些结果必须审核;Agent 在允许范围内处理不确定性。这就是混合架构——不是偏好,是行业试错后的**收敛终点**:能力可以一直长(九步),控制权必须留在确定性工程手里。

### 佐证:为什么我们也选混合架构(五特征对照表)
业务(API 接入、联调排查、单量分析)不是开放闲聊,而是**高频、确定、可审核的流程**:
| 业务特征 | 因此 Agent 需要 | 对应模块(后文详解) |
|---|---|---|
| 流程顺序稳定 | 不要每次重想流程 | Workflow |
| 字段规则明确 | 判断可校验 | Schema + Validator |
| 问题高度重复 | 复用专家经验 | Skill + 知识库 |
| 任务多轮推进 | 知道走到哪一步 | Task State |
| 凭证敏感、上线敏感 | 必须人工确认 | Audit + 人工审核 |

收尾(讲给业务同学的一句话):Agent 不是越自主越好;把大模型放进可控流程——LLM 负责理解和表达,Workflow / Schema / Skill / Task State / Memory / Audit 和人工审核负责稳定、可控、可复盘。
